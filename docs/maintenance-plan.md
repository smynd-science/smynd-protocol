# Maintenance Plan

This repository should stay active through real, small, reviewable work.

## Weekly loop

1. Pick one issue from the current milestone.
2. Create a branch.
3. Make a small change.
4. Add or update tests/docs.
5. Open a PR.
6. Merge after CI passes.
7. Update `CHANGELOG.md` if user-visible behavior changed.

## Do not create fake activity

Avoid empty commits, meaningless churn, or cosmetic updates that do not improve the project.

## Good recurring updates

- Add one test case.
- Improve one section of documentation.
- Close one issue with a real PR.
- Add one synthetic example.
- Improve one error message.
- Update one threat-model item.
